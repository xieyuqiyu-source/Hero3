# 将领系统安全隐患与漏洞排查报告

## 🚨 严重安全隐患

### 1. **将领配置参数无验证 - 可导致游戏经济崩溃**

**位置**: `go/internal/game/generals_config.go` + `go/internal/general/trait.go`

**问题描述**:
- 将领特性参数（`Params map[string]float64`）**完全没有服务端验证**
- GM 后台可以任意修改参数，运行时立即生效（`applyHeroConfigToGeneral` 每次读取存档都会调用）
- 参数可以设置为**任意值**，包括负数、极大值、超出范围值

**危害等级**: ⚠️ **极高**

**攻击场景**:
```json
// GM 可以设置如下参数导致游戏崩溃
{
  "meiren": {
    "captureRate": 999.0,      // 俘虏999倍兵力
    "captureMax": 999999999,   // 单次俘虏上亿兵
    "triggerChance": 999.0     // 触发概率999倍
  },
  "huogong": {
    "damagePercent": 1.0,      // 100%伤害 = 秒杀所有敌军
    "triggerChance": 1.0
  },
  "rende": {
    "reviveRate": 10.0,        // 复活1000%，战斗不仅不死还增兵
    "triggerChance": 1.0
  }
}
```

**代码证据**:
```go
// go/internal/general/trait.go
func (p Params) FloatOr(key string, def float64) float64 {
    if v, ok := p[key]; ok {
        return v  // ❌ 直接返回，没有任何验证
    }
    return def
}

// go/internal/general/traits/meiren.go
rate := p.FloatOr("captureRate", 0.1)
maxPerType := p.IntOr("captureMax", 1000)
// ❌ rate 可以是负数、可以 > 1、可以是 999
// ❌ maxPerType 可以是负数、可以是十亿
```

**修复建议**:
```go
// 1. 在 ParamField 中添加验证规则
type ParamField struct {
    Key         string  `json:"key"`
    Label       string  `json:"label"`
    Min         float64 `json:"min"`
    Max         float64 `json:"max"`
    // 添加：
    Required    bool    `json:"required"`
}

// 2. 在 Params 方法中添加边界检查
func (p Params) FloatWithBounds(key string, def, min, max float64) float64 {
    v := p.FloatOr(key, def)
    if v < min {
        return min
    }
    if v > max {
        return max
    }
    return v
}

// 3. 在 UpdateGeneralsConfig 时验证所有参数
func ValidateGeneralsConfig(cfg GeneralsConfig) error {
    for heroID, hero := range cfg.Heroes {
        for _, trait := range hero.Traits {
            traitDef, ok := general.Get(trait.TraitID)
            if !ok {
                return fmt.Errorf("unknown trait: %s", trait.TraitID)
            }
            schema := traitDef.ParamSchema()
            for _, field := range schema {
                value, ok := trait.Params[field.Key]
                if field.Required && !ok {
                    return fmt.Errorf("missing required param %s", field.Key)
                }
                if ok && (value < field.Min || value > field.Max) {
                    return fmt.Errorf("param %s out of range [%f, %f]", field.Key, field.Min, field.Max)
                }
            }
        }
    }
    return nil
}
```

### 2. **将领选择无归属校验 - 可跨阵营选择将领**

**位置**: `go/internal/game/service.go::CreatePlayer`

**问题描述**:
- 创建玩家时，服务端**只验证 generalID 是否存在于阵营配置中**
- **没有验证 `generals.json` 中将领的 `faction` 字段**
- 攻击者可以修改前端请求，选择其他阵营的将领

**危害等级**: ⚠️ **高**

**攻击场景**:
```javascript
// 前端发送请求
POST /api/v1/game/create
{
  "accountId": "acc_xxx",
  "nickname": "作弊玩家",
  "faction": "wei",      // 选择魏国
  "generalId": "zhouyu"  // 但使用吴国的周瑜（火攻特性）
}
```

**代码证据**:
```go
// go/internal/game/service.go::CreatePlayer
func (s *Service) CreatePlayer(..., generalID string) (string, GameState, error) {
    factions := GetFactionsConfig()
    fc, factionExists := factions[faction]
    if !factionExists {
        return "", GameState{}, ErrInvalidGeneral
    }
    valid := false
    for _, g := range fc.Generals {
        if g.ID == generalID {  // ❌ 只检查 ID 是否在列表里
            valid = true
            break
        }
    }
    // ❌ 没有检查 generals.json 中该将领的 faction 字段
}
```

**修复建议**:
```go
func (s *Service) CreatePlayer(..., generalID string) (string, GameState, error) {
    // 1. 验证阵营配置
    factions := GetFactionsConfig()
    fc, factionExists := factions[faction]
    if !factionExists {
        return "", GameState{}, ErrInvalidGeneral
    }
    
    // 2. 验证将领在阵营列表中
    valid := false
    for _, g := range fc.Generals {
        if g.ID == generalID {
            valid = true
            break
        }
    }
    if !valid {
        return "", GameState{}, ErrInvalidGeneral
    }
    
    // 3. 双重验证：检查将领配置中的阵营
    generalsCfg := GetGeneralsConfig()
    hero, ok := generalsCfg.Heroes[generalID]
    if !ok || !hero.Enabled {
        return "", GameState{}, ErrInvalidGeneral
    }
    if hero.Faction != faction {  // ✅ 新增：阵营一致性检查
        return "", GameState{}, ErrInvalidGeneral
    }
    
    // ... 继续创建玩家
}
```

### 3. **Admin API 无速率限制 - 可被暴力攻击**

**位置**: `go/internal/api/router.go` + `go/internal/auth/auth.go`

**问题描述**:
- Admin Token 是静态字符串，**永不过期**
- Admin API **没有任何速率限制**
- 可以暴力破解 Admin Token，或者泄露后无限制调用

**危害等级**: ⚠️ **高**

**攻击场景**:
```bash
# 暴力破解 Admin Token
for token in $(cat wordlist.txt); do
  curl -H "X-Admin-Token: $token" \
       http://server/api/v1/admin/generals-config
done

# Token 泄露后无限修改配置
while true; do
  curl -H "X-Admin-Token: leaked_token" \
       -X PUT http://server/api/v1/admin/generals-config \
       -d '{"enabled":false}'
  sleep 0.1
done
```

**代码证据**:
```go
// go/internal/auth/auth.go
func AuthMiddleware(cfg Config, publicPaths []string) func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            if adminToken := r.Header.Get("X-Admin-Token"); adminToken != "" {
                if cfg.AdminToken != "" && adminToken == cfg.AdminToken {
                    ctx := context.WithValue(r.Context(), ctxIsAdmin, true)
                    next.ServeHTTP(w, r.WithContext(ctx))  // ❌ 直接放行，无限制
                    return
                }
            }
        })
    }
}
```

**修复建议**:
```go
// 1. 添加 IP 速率限制
type AdminRateLimiter struct {
    mu       sync.Mutex
    attempts map[string][]time.Time // IP -> 尝试时间列表
}

func (rl *AdminRateLimiter) Allow(ip string, maxAttempts int, window time.Duration) bool {
    rl.mu.Lock()
    defer rl.mu.Unlock()
    
    now := time.Now()
    attempts := rl.attempts[ip]
    
    // 清理过期记录
    var validAttempts []time.Time
    for _, t := range attempts {
        if now.Sub(t) <= window {
            validAttempts = append(validAttempts, t)
        }
    }
    
    if len(validAttempts) >= maxAttempts {
        return false
    }
    
    validAttempts = append(validAttempts, now)
    rl.attempts[ip] = validAttempts
    return true
}

// 2. 添加 Token 轮换机制
type AdminTokenStore struct {
    tokens    map[string]time.Time // token -> 创建时间
    tokenTTL  time.Duration        // token 有效期
}

// 3. 添加操作审计日志
func logAdminOperation(ip, operation string, success bool) {
    log.Printf("[AUDIT] IP=%s Operation=%s Success=%v", ip, operation, success)
}
```

## ⚠️ 中危漏洞

### 4. **将领属性叠加无上限 - 可堆叠到整数溢出**

**位置**: `go/internal/game/model.go::applyHeroConfigToGeneral`

**问题描述**:
- 将领属性通过 `addGeneralAttribute` 不断累加
- **没有上限检查**，理论上可以累加到 `float64` 最大值或溢出
- 虽然当前只有等级、四维、固定属性三个来源，但未来扩展可能失控

**代码证据**:
```go
func addGeneralAttribute(attrs map[string]float64, key string, value float64) {
    if attrs == nil || key == "" || value == 0 {
        return
    }
    attrs[key] += value  // ❌ 无限累加，无上限
}
```

**修复建议**:
```go
const (
    MaxAttributeValue = 10.0  // 属性上限 1000%
)

func addGeneralAttribute(attrs map[string]float64, key string, value float64) {
    if attrs == nil || key == "" || value == 0 {
        return
    }
    newValue := attrs[key] + value
    if newValue > MaxAttributeValue {
        attrs[key] = MaxAttributeValue  // ✅ 限制上限
    } else {
        attrs[key] = newValue
    }
}
```

### 5. **美人计俘虏兵力可超过军队上限**

**位置**: `go/internal/general/traits/meiren.go` + `go/internal/game/service_combat.go`

**问题描述**:
- 美人计俘虏敌方兵力后，直接添加到 `CapturedToArmy`
- **没有检查玩家军队容量上限**
- 可以通过战斗突破军队上限，导致游戏平衡崩溃

**代码证据**:
```go
// go/internal/general/traits/meiren.go
for i := range c.Defender.Units {
    unit := &c.Defender.Units[i]
    captured := int(float64(unit.Count) * rate)
    // ...
    dest[unit.ID] += captured  // ❌ 无限累加，不检查容量
}

// go/internal/game/service_combat.go
for unitType, count := range beforeCtx.CapturedToArmy {
    state.Army = appendOrUpdateArmy(state.Army, unitType, count)
    // ❌ appendOrUpdateArmy 也不检查容量
}
```

**修复建议**:
```go
// 在战斗后应用俘虏时检查容量
func applyBattleCaptures(state *GameState, captures map[string]int) error {
    currentTotal := totalArmyCount(state.Army)
    captureTotal := 0
    for _, count := range captures {
        captureTotal += count
    }
    
    // 计算容量
    capacity := calculateArmyCapacity(state.Buildings)
    
    if currentTotal + captureTotal > capacity {
        // 选择策略：
        // 1. 拒绝全部俘虏
        // 2. 部分俘虏（按比例）
        // 3. 超出部分进驻防
        excessCaptures := apportionExcessToGarrison(captures, capacity - currentTotal)
        for unitType, count := range excessCaptures {
            state.Garrison = appendOrUpdateArmy(state.Garrison, unitType, count)
        }
    } else {
        for unitType, count := range captures {
            state.Army = appendOrUpdateArmy(state.Army, unitType, count)
        }
    }
    return nil
}
```

### 6. **特性触发概率可被设置为负数或超过1**

**位置**: `go/internal/general/traits/*.go`

**问题描述**:
- `triggerChance` 参数没有在使用前验证范围
- 虽然 `ParamSchema` 定义了 `Min: 0, Max: 1`，但实际代码中**没有强制执行**
- 负数概率可能导致逻辑错误或永远不触发

**代码证据**:
```go
// go/internal/general/traits/meiren.go
chance := p.FloatOr("triggerChance", 1.0)
if chance < 1.0 && rand.Float64() > chance {  // ❌ 如果 chance 是负数或 >1 呢？
    return
}
```

**修复建议**:
```go
// 在特性代码中添加边界检查
func getValidatedChance(p general.Params, key string, def float64) float64 {
    chance := p.FloatOr(key, def)
    if chance < 0 {
        return 0
    }
    if chance > 1 {
        return 1
    }
    return chance
}

// 使用
chance := getValidatedChance(p, "triggerChance", 1.0)
```

### 7. **周瑜火攻可导致整数越界**

**位置**: `go/internal/general/traits/huogong.go`

**问题描述**:
- 火攻计算 `extra := int(math.Floor(float64(loss.Count) * damagePct))`
- 如果 `loss.Count` 很大（如 2^30），`damagePct` 也很大（如 100），结果可能超过 `int` 最大值
- Go 中 `int` 在 32 位系统上只有 2^31-1

**代码证据**:
```go
extra := int(math.Floor(float64(loss.Count) * damagePct))
// ❌ 如果 loss.Count = 2^30, damagePct = 10, 结果超过 int32
loss.Losses += extra
```

**修复建议**:
```go
extra := int(math.Floor(float64(loss.Count) * damagePct))
if extra < 0 {  // ✅ 溢出检测
    extra = loss.Count  // 全灭
}
loss.Losses += extra
if loss.Losses < 0 {  // ✅ 二次溢出检测
    loss.Losses = loss.Count
}
```

## ℹ️ 低危问题

### 8. **将领经验公式可导致升级卡死**

**位置**: `go/internal/game/service_general.go::generalExpRequiredForLevel`

**问题描述**:
- 使用四次方公式：`n^2 * 50 + n^4 * 500`
- 在高等级时增长极快，可能导致升级需要天文数字经验
- 100 级需要经验约 `500 * 99^4 = 4,802,440,500`（48亿）

**代码证据**:
```go
func generalExpRequiredForLevel(level int) int {
    n := level - 1
    n2 := n * n
    n4 := n2 * n2
    return generalExpQuadraticFactor*n2 + generalExpQuarticFactor*n4
}
```

**建议**:
- 数值策划评估高等级经验需求是否合理
- 考虑使用分段函数或对数曲线平滑增长

### 9. **旧存档兼容代码可能被滥用**

**位置**: `go/internal/game/service.go::GetState`

**问题描述**:
- 如果 `state.General == nil`，自动分配默认将领
- 虽然是为了兼容旧存档，但可能被玩家利用：
  - 手动删除存档中的 `general` 字段
  - 重新进入游戏获得默认将领（可能是不同的将领）

**代码证据**:
```go
// 旧存档没有将领数据时，根据阵营分配默认将领
if state.General == nil && state.Player.Faction != "" {
    defaultGenerals := map[string]string{
        "wei": "caocao",
        "shu": "liubei",
        "wu":  "sunquan",
    }
    if gid, ok := defaultGenerals[state.Player.Faction]; ok {
        state.General = newGeneral(state.Player.Faction, gid)
        changed = true
    }
}
```

**修复建议**:
```go
// 添加标志位，只对真正的旧存档执行一次
if state.General == nil && state.Player.Faction != "" {
    // 检查是否是首次迁移
    if state.Player.CreatedAt.Before(generalSystemReleaseDate) {
        // 只对老玩家执行迁移
        defaultGenerals := map[string]string{...}
        state.General = newGeneral(state.Player.Faction, gid)
    } else {
        // 新玩家没有将领 = 数据异常，拒绝

}
```
