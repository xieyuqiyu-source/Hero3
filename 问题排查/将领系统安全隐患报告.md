#### **1. 【高危】参数验证存在漏洞**

**问题**：虽然使用了 `bounded()` 函数，但只在**读取时**做边界检查，**配置保存时没有验证**。

```go
// 当前代码
chance := bounded(p.FloatOr("baseChance", 0.08), 0, 1) +
    (ratio-1)*bounded(p.FloatOr("chancePerRatio", 0.04), 0, 1)
```

**攻击场景**：
```json
{
  "zhangliao": {
    "traits": [{
      "traitId": "weizhenxiaoyao",
      "params": {
        "baseChance": -100,      // 负数会被 bounded 修正为 0，但暴露了验证漏洞
        "maxChance": 999,        // 会被修正为 1，但没有告警
        "chancePerRatio": 100,   // 会被修正为 1，导致敌我口粮比 1.01 就满概率
        "maxSuppressRate": 0.95  // 可以震慑 95% 的守军
      }
    }]
  }
}
```

**影响**：虽然 `bounded()` 防止了崩溃，但**没有在配置更新时验证和拒绝非法值**，GM 可以设置极端值而不知道被修正了。

**修复建议**：
```go
// 在 go/internal/game/service_admin.go::UpdateGeneralsConfig 中添加验证
func ValidateWeizhenxiaoyaoParams(params map[string]float64) error {
    checks := map[string][2]float64{
        "baseChance":       {0, 1},
        "chancePerRatio":   {0, 1},
        "maxChance":        {0, 1},
        "baseSuppressRate": {0, 1},
        "suppressPerRatio": {0, 1},
        "maxSuppressRate":  {0, 1},
    }
    
    for key, bounds := range checks {
        if val, ok := params[key]; ok {
            if val < bounds[0] || val > bounds[1] {
                return fmt.Errorf("weizhenxiaoyao.%s=%f out of range [%f, %f]", 
                    key, val, bounds[0], bounds[1])
            }
        }
    }
    
    // 逻辑一致性检查
    if params["maxChance"] < params["baseChance"] {
        return fmt.Errorf("maxChance must >= baseChance")
    }
    if params["maxSuppressRate"] < params["baseSuppressRate"] {
        return fmt.Errorf("maxSuppressRate must >= baseSuppressRate")
    }
    
    return nil
}
```

#### **2. 【中危】震慑兵力可能为 0，但仍记录触发**

**问题**：当震慑率极低时，`suppressDefenderUnits` 可能所有单位都震慑 0 个兵，但代码仍会记录特性触发。

```go
func (w *WeizhenXiaoyao) beforeBattle(ctx general.EventContext, p general.Params) {
    // ... 计算 suppressRate
    suppressed := suppressDefenderUnits(c.Defender, suppressRate)
    totalSuppressed := 0
    for _, count := range suppressed {
        totalSuppressed += count
    }
    if totalSuppressed <= 0 {
        return  // ✅ 已经检查了，但有逻辑缺陷
    }
}

func suppressDefenderUnits(army *combat.Army, rate float64) map[string]int {
    suppressed := map[string]int{}
    for i := range army.Units {
        unit := &army.Units[i]
        if unit.Count <= 0 {
            continue
        }
        count := int(math.Floor(float64(unit.Count) * rate))
        if count <= 0 && rate > 0 {
            count = 1  // ❌ 问题：即使震慑率 0.001，也至少震慑 1 个
        }
        // ...
    }
}
```

**影响**：
- 即使 `suppressRate = 0.001`（0.1%），每种兵至少震慑 1 个
- 如果敌方有 10 种兵，每种 10 个，震慑率 0.001 应该震慑 `100 * 0.001 = 0.1` 个（向下取整为 0）
- 但实际会震慑 10 个（每种 1 个），**放大了 100 倍效果**

**修复建议**：
```go
func suppressDefenderUnits(army *combat.Army, rate float64) map[string]int {
    suppressed := map[string]int{}
    totalDefenders := 0
    for _, unit := range army.Units {
        totalDefenders += unit.Count
    }
    
    // 先计算总共应该震慑多少兵
    totalToSuppress := int(math.Floor(float64(totalDefenders) * rate))
    if totalToSuppress <= 0 {
        return suppressed  // ✅ 不震慑任何兵
    }
    
    // 按比例分配到各单位
    remaining := totalToSuppress
    for i := range army.Units {
        unit := &army.Units[i]
        if unit.Count <= 0 || remaining <= 0 {
            continue
        }
        
        // 按兵力占比分配
        proportion := float64(unit.Count) / float64(totalDefenders)
        count := int(math.Floor(float64(totalToSuppress) * proportion))
        
        // 处理余数：如果还有剩余且当前单位还有兵，至少震慑 1 个
        if count == 0 && remaining > 0 {
            count = 1
        }
        
        if count > unit.Count {
            count = unit.Count
        }
        if count > remaining {
            count = remaining
        }
        
        unit.Count -= count
        suppressed[unit.ID] += count
        remaining -= count
    }
    
    return suppressed
}
```

#### **3. 【中危】口粮计算未考虑 Upkeep = 0 的单位**

**问题**：代码中有 `if upkeep <= 0 { upkeep = 1 }` 的兜底逻辑，但这可能导致平衡问题。

```go
func armyFood(units []combat.Unit) float64 {
    var total float64
    for _, unit := range units {
        if unit.Count <= 0 {
            continue
        }
        upkeep := unit.Upkeep
        if upkeep <= 0 {
            upkeep = 1  // ❌ 假设所有 0 口粮单位都是 1
        }
        total += float64(unit.Count * upkeep)
    }
    return total
}
```

**影响**：
- 如果有特殊兵种（如幻影、召唤物）设计为 `upkeep = 0`
- 这里会强制当作 `upkeep = 1`，导致口粮比计算错误
- 可能让玩家通过堆 0 口粮单位来触发"以少打多"

**建议**：
```go
func armyFood(units []combat.Unit) float64 {
    var total float64
    for _, unit := range units {
        if unit.Count <= 0 {
            continue
        }
        upkeep := unit.Upkeep
        if upkeep < 0 {
            upkeep = 0  // ✅ 负数是配置错误，记为 0
        }
        // ✅ 如果真的是 0，就按 0 计算（可能是设计意图）
        total += float64(unit.Count * upkeep)
    }
    return total
}

// 并在触发条件中添加最低口粮要求
if attackerFood <= 10 || defenderFood <= attackerFood {  // ✅ 至少 10 口粮才能触发
    return
}
```
