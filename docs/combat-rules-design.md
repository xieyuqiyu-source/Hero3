# 战斗规则设计文档

最后更新：2026-05-24

## 1. 设计目标

- 攻防分离：进攻方只算攻击力，防守方只算防御力
- 兵种配比有意义：步兵多打高步防亏，骑兵多打高骑防亏
- 两种战斗模式：攻击（决战）和掠夺（试探）
- 规则可配置：GM 可切换不同战斗规则集
- 后续可扩展：将领加成、城墙加成、词条加成等通过乘数接入

## 2. 核心公式

### 2.1 进攻方计算

将进攻方所有兵种按类型分为步兵类和骑兵类：

```
A1 = Σ(步兵类兵种的 attack × 数量)    // 步兵攻击总和
A2 = Σ(骑兵类兵种的 attack × 数量)    // 骑兵攻击总和
a  = A1 + A2                           // 进攻总攻击力
```

兵种分类规则：
- category = "infantry" 或 "siege" 或 "special" → 计入 A1（步兵类）
- category = "cavalry" → 计入 A2（骑兵类）

### 2.2 防守方计算

防守方的有效防御按进攻方的步/骑攻击比例加权：

```
D1 = Σ(所有守军的 infantryDefense × 数量)   // 对步防御总和
D2 = Σ(所有守军的 cavalryDefense × 数量)    // 对骑防御总和
b  = (A1 × D1 + A2 × D2) / (A1 + A2)       // 加权有效防御
```

含义：如果进攻方全是步兵（A2=0），则 b=D1（只看步防）；如果全是骑兵（A1=0），则 b=D2（只看骑防）。混编则按比例折算。

### 2.3 城墙加成（仅 PvP）

攻击玩家城池时，防守方有效防御乘以城墙系数：

```
B = b × c^城墙等级
```

各阵营城墙系数：
- 魏：c = 1.03
- 吴：c = 1.025
- 蜀：c = 1.02

城墙等级加成一览（%）：

| Lv | 魏 | 吴 | 蜀 |
|----|----|----|-----|
| 1 | 3% | 2% | 2% |
| 5 | 16% | 13% | 10% |
| 10 | 34% | 28% | 22% |
| 15 | 56% | 45% | 35% |
| 20 | 81% | 64% | 49% |

注：攻击 NPC 城池时不计算城墙加成。

### 2.4 胜负判定

```
if a > b  → 进攻方胜
if a < b  → 防守方胜
if a = b  → 平局
```

## 3. 损失计算

常数 c = 1.422

### 3.1 攻击模式（决战）

#### a > b（进攻方胜）
```
进攻方损失比例 = (b/a)^1.422
防守方损失比例 = 100%（全灭）
```

#### a < b（防守方胜）
```
进攻方损失比例 = 100%（全灭）
防守方损失比例 = (a/b)^1.422
```

#### a = b（平局）
```
进攻方损失比例 = 100%
防守方损失比例 = 100%（同归于尽）
```

### 3.2 掠夺模式（试探）

#### a > b（进攻方胜）
```
ratio = (b/a)^1.422
进攻方损失比例 = ratio / (1 + ratio)
防守方损失比例 = 1 / (1 + ratio)
```

#### a < b（防守方胜）
```
ratio = (a/b)^1.422
进攻方损失比例 = 1 / (1 + ratio)
防守方损失比例 = ratio / (1 + ratio)
```

#### a = b（平局）
```
进攻方损失比例 = 50%
防守方损失比例 = 50%
```

### 3.3 掠夺模式特性

- 双方损失比例之和恒等于 100%
- 无论胜负，双方都不会全灭
- 适合试探性攻击和资源掠夺

## 4. 损失分配

损失比例确定后，按比例分配到各兵种：

```
每个兵种损失 = floor(该兵种数量 × 损失比例)
```

所有兵种取相同的损失比例（按比例均摊），最终结果向下取整。

取整后可能有余数（总损失略少于理论值），余数按各兵种小数部分从大到小补 1，直到总损失达到理论值。

## 5. 掠夺资源计算

战斗胜利后（仅进攻方胜时触发掠夺）：

```
存活部队运载量 = Σ(存活兵种数量 × carryCapacity)
可掠夺总量 = min(存活部队运载量, 防守方当前资源总量)
各资源分配 = 按防守方各资源占比按比例分配
```

分配算法：
1. 计算防守方各资源占总资源的比例
2. 按比例分配运载量到各资源
3. 每种资源实际掠夺量 = min(分配量, 该资源实际存量)
4. 向下取整

## 6. 战斗流程

### 6.1 PvE（攻击 NPC 城池）

```
1. 玩家选择目标 NPC 城池
2. 选择战斗模式（攻击/掠夺）
3. 选择出征兵力
4. 行军（1-5 分钟）
5. 到达 → 结算 NPC 当前资源和守军（惰性结算）
6. 战斗计算（不计城墙）
7. 更新 NPC 状态（扣减资源和守军）
8. 计算掠夺资源（如果进攻方胜）
9. 返程行军
10. 到达 → 资源入库，部队归营
11. 生成战报
```

### 6.2 PvP（攻击玩家城池，后续）

```
1-4. 同上
5. 到达 → 结算对方当前资源
6. 战斗计算（计算城墙加成）
7-11. 同上
```

## 8. 验算示例

### 示例 1：进攻方碾压（攻击模式）

玩家派 200 青州军（步兵 attack=8）攻击 NPC 守军 50 青州军：

```
A1 = 200 × 8 = 1600, A2 = 0
a = 1600

D1 = 50 × 7 = 350, D2 = 50 × 10 = 500
b = (1600 × 350 + 0 × 500) / 1600 = 350

a > b → 进攻方胜
进攻方损失 = (350/1600)^1.422 = 0.2188^1.422 ≈ 0.112 → 11.2%
→ 损失 200 × 0.112 = 22 人，存活 178 人
防守方全灭：50 人全部阵亡
```

### 示例 2：势均力敌（攻击模式）

玩家派 100 青州军攻击 NPC 守军 80 青州军：

```
A1 = 100 × 8 = 800, A2 = 0
a = 800

D1 = 80 × 7 = 560, D2 = 80 × 10 = 800
b = (800 × 560 + 0) / 800 = 560

a > b → 进攻方胜
进攻方损失 = (560/800)^1.422 = 0.7^1.422 ≈ 0.579 → 57.9%
→ 损失 100 × 0.579 = 57 人，存活 43 人
防守方全灭：80 人全部阵亡
```

### 示例 3：掠夺模式

玩家派 100 青州军掠夺 NPC 守军 80 青州军：

```
a = 800, b = 560（同上）
ratio = (560/800)^1.422 = 0.579

进攻方损失 = 0.579 / (1 + 0.579) = 0.367 → 36.7%
→ 损失 100 × 0.367 = 36 人，存活 64 人

防守方损失 = 1 / (1 + 0.579) = 0.633 → 63.3%
→ 损失 80 × 0.633 = 50 人，存活 30 人

掠夺：64 人 × carryCapacity(80) = 5120 运载量
```

### 示例 4：混编进攻

玩家派 80 青州军（步兵 attack=8）+ 20 骁骑营（骑兵 attack=24）攻击 NPC 守军 100 青州军：

```
A1 = 80 × 8 = 640（步兵攻击）
A2 = 20 × 24 = 480（骑兵攻击）
a = 640 + 480 = 1120

D1 = 100 × 7 = 700（守军步防）
D2 = 100 × 10 = 1000（守军骑防）
b = (640 × 700 + 480 × 1000) / 1120 = (448000 + 480000) / 1120 = 828.6

a > b → 进攻方胜
进攻方损失 = (828.6/1120)^1.422 = 0.74^1.422 ≈ 0.618 → 61.8%
→ 青州军损失 80 × 0.618 = 49 人
→ 骁骑营损失 20 × 0.618 = 12 人
防守方全灭
```

注意：因为守军骑防(10)高于步防(7)，混编骑兵反而让 b 变大了。如果全用步兵 b=700，混编后 b=828.6，损失更大。这就是"配兵有讲究"的体现。

## 9. 规则配置结构

```json
{
  "activeCombatRules": {
    "pve_attack": "official_attack",
    "pve_plunder": "official_plunder",
    "pvp_attack": "official_attack",
    "pvp_plunder": "official_plunder"
  },
  "rules": {
    "official_attack": {
      "id": "official_attack",
      "name": "官方攻击规则",
      "mode": "attack",
      "exponent": 1.422,
      "equalResult": "mutual_destruction",
      "lossDistribution": "proportional",
      "defenseFormula": "weighted"
    },
    "official_plunder": {
      "id": "official_plunder",
      "name": "官方掠夺规则",
      "mode": "plunder",
      "exponent": 1.422,
      "equalResult": "half_each",
      "lossDistribution": "proportional",
      "defenseFormula": "weighted"
    }
  },
  "wallConfig": {
    "wei": { "base": 1.03 },
    "shu": { "base": 1.02 },
    "wu": { "base": 1.025 }
  }
}
```

GM 可通过修改 `activeCombatRules` 切换不同场景使用的规则，也可以新增规则到 `rules` 中。

## 10. 后续扩展点

以下内容当前不实现，但架构预留接入点：

| 扩展 | 接入方式 |
|------|----------|
| 将领攻击加成 | 计算 a 后乘以将领 attackBonus |
| 将领防御加成 | 计算 b 后乘以将领 defenseBonus |
| NPC 词条加成 | 计算 D1/D2 时乘以词条 buffs |
| 阵营特性 | 计算 a/b 后乘以阵营 militaryBonus |
| 城墙 | 计算 b 后乘以城墙系数（仅 PvP） |
| 新规则集 | 在 rules 配置中新增，切换 activeCombatRules |

所有加成通过乘数叠加，不修改核心公式逻辑。
